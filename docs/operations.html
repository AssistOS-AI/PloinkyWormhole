Operations and Dashboard

The runtime now includes explicit operator-facing observability: public liveness/readiness, authenticated metrics and event streams, manual blocklist management, and a browser dashboard shell that works with header-based admin authentication.

Public operational endpoints

EndpointPurpose
GET /healthLiveness and coarse attack-level signal for load balancers.
GET /readyProduction-readiness checks and warnings, returns 503 when degraded.
GET /limitsPublished resource bounds and hardening knobs.

Admin observability

The admin JSON APIs are protected by x-admin-token. /admin/metrics exposes uptime, attack level, route counters, queue sizes, cache sizes, effective blocklist counts, active auto-blocks, and top offenders. /admin/events exposes the bounded recent event buffer.

Dashboard shell

/admin/dashboard serves an HTML shell with no embedded live data. The operator enters the admin token locally in the browser; the page then fetches metrics, events, and blocklist data using request headers instead of URL parameters.

Manual blocklist workflow

Operators can add or remove persistent manual blocks for IPs, DIDs, and domains using /admin/blocklist. Manual entries survive restart through the admin-state JSON file. Temporary auto-blocks remain visible in the same view and can also be cleared through the same endpoint.

Attack-level guidance

The runtime derives normal, elevated, and under_attack states from recent offense activity and active auto-blocks. This is meant to guide operators and dashboards, not to replace external logging or incident workflows.
