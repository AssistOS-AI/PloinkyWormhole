The API mirrors three protocol concepts plus explicit operational and admin surfaces.

Protocol constraints

Local agent writes must use a local from_did on /intent, /intent-response, and /signal. Forwarded writes are distinguished only by relay headers; the body model does not change.

Responses must target an existing local canonical intent when the responder is local. Signals must be bounded in size and time, and they are accepted only when the server can prove the local canonical intent is already accepted.

Admin contract

Admin APIs accept the token only in x-admin-token. The dashboard shell asks the operator to paste the token locally in the browser and then calls the JSON APIs with headers. Tokens must never appear in query strings.
